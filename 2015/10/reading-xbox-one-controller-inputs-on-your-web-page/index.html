Reading Xbox One Controller Inputs on Your Web Page

Oct 23rd 2015 (Comments)

If you've ever tried to browse any website on the Xbox One, you'll quickly find out that it's ok, but not the same navigating a native application or game. I.E. on the Xbox One has a mouse and you can zoom around, navigate slowly through pages.

But what if you could design your website to navigate and operate just like a native xbox application?

I found a great write up of the navigator.getGamepads() api that the native I.E. browser implements here.

It took a sample page he had, threw it into a jsbin, modified it with following changes.

  1. Add the following toastr library <script type="text/javascript" src="https://cdnjs.cloudflare.com/ajax/libs/toastr.js/2.1.2/toastr.min.js"></script>
  2. Add the following debug code
document.addEventListener('keydown', function(event){
    toastr.success(event.keyCode + " - " + event.keyIdentifier );
});
  1. Save the jsbin
  2. Walk over to my personal family Xbox One, open the jsbin page one I.E.
  3. Profit... (profit by learning what all the keyCode values are for each button on the controller)

THIS WAS AWESOME...

In fact I found out that these same values are confirmed in the WinJS source code...
