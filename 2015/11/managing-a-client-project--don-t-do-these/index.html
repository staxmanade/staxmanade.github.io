Managing a client project? Don't do These

Nov 11th 2015 (Comments)

What not to say

  • That is not out of scope = We should add it to the backlog
  • That is not the current priority = Do you want that to be the top priority
  • Without the API we are blocked = We can investigate what the data will most likely look like then start implementation when the API is ready
  • That’s a new feature = We should investigate and add it to the backlog
  • We do not work weekends = We will look at the schedule and determine where we will fit this work
  • We do not have additional resources for that request = What is the drive for this request and where does if fall in priority
  • We do not have additional time in the schedule = Do you have more money to give us
  • You can not have that earlier than expected = We will what is possible and get back to you
  • No Jason can not work on ATV and GCM at the same time = Yes Jason can work on both at the same time
  • Implementing that SDK will take 3 weeks longer than expected = The lift is large we will need to assess current capacity and determine when/if we can fit this in
  • Without designs we are blocked = Let’s review the current design and whiteboard design ups and agree to changes and who will delivery updated designs
